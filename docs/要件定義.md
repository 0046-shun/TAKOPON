 # 要件定義
 
 ## 目的
 たこやきをキッチンカーで販売するお店のWebサイトを作成する。
 
 ## 対象ユーザー
 - 全世界ユーザー
 - 日本語版・英語版の多言語対応
 
## 掲載内容（公開ページ）
- お店の説明
- メニュー
- 出店情報
  - 出店場所
  - Googleマップ埋め込み（各出店ごと）
- お知らせ（ニュース）
 
 ## 機能
 - 管理者による出店情報の編集
  - メニューはコーダーが更新対応

## 管理画面要件（出店情報）
- 一覧 / 新規 / 編集 / 削除
- 下書き / 公開の切替
- 並び順の変更
- 公開期間の自動非表示
- 公開予約（開始日時で自動公開）
- 複製（過去出店情報のコピーから新規作成）
- 画像アップロード（出店ごとの写真）
- 多言語入力（日本語 / 英語を管理画面で別入力）
- 入力バリデーション（期間の逆転禁止、必須チェックなど）

## 管理画面要件（お知らせ）
- 管理画面で更新対象に含める
- 項目：タイトル / 本文 / 日付 / リンク
- 公開期間（開始 / 終了）
- リンクは外部URLのみ

## 掲載内容の詳細
- お店の説明：こだわり / ストーリー / 写真
  - 店舗名
  - コンセプト / キャッチコピー
  - 主な特徴（生地 / ソース / 食材など）
  - 連絡先（電話を公開 / メールは掲載しない）
  - 営業時間（キッチンカーのため目安・注記）
  - SNSリンク（Instagram）
  - 受賞歴・メディア掲載（任意）
- メニュー：商品名 / 価格 / 説明 / 写真
  - 価格は税込/税抜の表記
  - 掲載数は多くない想定
  - オーダー方法の案内は不要（口頭対応）

## 画面構成・導線（イメージ準拠）
- 縦スクロール中心の1ページ構成
- 画面上部に簡易ナビ（各セクションへアンカー移動）
- セクション例：お店の説明 / お知らせ / メニュー / 出店情報

## 出店情報の要件
- 必須項目：期間（開始日・終了日）、住所、地図（Googleマップ埋め込み）、備考
- 備考はテキストのみ（例：雨天中止、営業時間補足、支払い方法など）
- 出店写真は任意

## 権限・運用
- 管理者は1名のみ
- 管理画面のアクセスはBasic認証（ID / パスワード）

## デザイン要件（添付イメージ準拠）
- ビビッドな単色背景（黄色基調）と黒系テキストで高コントラスト
- セクション間を細い罫線で区切るミニマル構成
- メニューは写真を大きく配置し、説明は短く読みやすく
- 余白を多めに取り、縦方向のリズムを重視

## 対応端末・表示要件
- スマホ対応（最優先）
- PC / タブレットでもレイアウトが崩れないレスポンシブ対応
- 言語切替は日本語 / 英語の2言語
  - ページ内トグル（ヘッダの言語スイッチ）

## フッター
- フッターは設けない
